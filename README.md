# ROC IO: أداة سطر الأوامر الذكية لتحليل الأكواد

![ROC IO Logo Placeholder](https://via.placeholder.com/800x200?text=ROC+IO+-+Smart+Code+Analysis+CLI)

**ROC IO** هي أداة قوية ومفتوحة المصدر لسطر الأوامر (CLI) مصممة لمساعدة المطورين، من المبتدئين إلى الخبراء، في فحص مشاريعهم البرمجية واكتشاف الأخطاء المحتملة (Bugs) أو المشكلات المتعلقة بأفضل الممارسات (Best Practices) في الكود. تعتمد الأداة على "بوتات" تحليل ذكية تعمل محليًا على ملفات مشروعك، مما يضمن السرية والسرعة.

تم بناء ROC IO باستخدام **Node.js** ومكتبات قوية مثل **Commander** و **Inquirer** لتوفير تجربة تفاعلية وسلسة في الترمينال (Terminal/CMD).

## الميزات الرئيسية

*   **تحليل محلي وسريع:** تعمل البوتات على ملفات مشروعك مباشرة دون الحاجة لرفعها إلى أي خوادم خارجية.
*   **اكتشاف الأخطاء المحتملة:** تركز على تحديد الأخطاء المحتملة، مشكلات الأمان، وحالات الكود التي تخالف أفضل الممارسات.
*   **تجربة مستخدم سلسة:** أوامر بسيطة وواجهة تفاعلية عبر سطر الأوامر.
*   **دعم متعدد المنصات:** تعمل بكفاءة على أنظمة Linux، macOS، و Windows.
*   **مفتوحة المصدر:** يمكن للمجتمع المساهمة في تطوير البوتات وإضافة قواعد تحليل جديدة.

## التثبيت

ROC IO هي أداة Node.js، لذا يجب التأكد من تثبيت Node.js (الإصدار 18 أو أحدث) على نظامك.

لتثبيت الأداة عالميًا (Global) لتكون متاحة من أي مكان في الترمينال:

```bash
npm install -g roc-io
# أو باستخدام pnpm
pnpm install -g roc-io
```

## الاستخدام

تتكون دورة عمل ROC IO من خطوتين رئيسيتين: الإعداد (Setup) والتشغيل (Run).

### 1. الإعداد: `rock-io setup`

يجب تشغيل هذا الأمر مرة واحدة لكل مشروع. يقوم الأمر بتهيئة ملف إعداد محلي (`.roc-io-config.json`) ويسمح لك باختيار الملفات والمجلدات التي تريد أن تقوم البوتات بتحليلها.

```bash
cd /path/to/your/project
rock-io setup
```

**ملاحظة:** يفضل أن تكون الملفات غير مضغوطة لتمكين البوتات من قراءة محتواها وتحليله بشكل صحيح.

### 2. التشغيل: `rock-io run`

بعد الإعداد، يمكنك تشغيل التحليل. هذا الأمر سيقوم بتشغيل البوتات على الملفات المحددة في ملف الإعداد.

#### أ. تشغيل لوحة التحكم (Dashboard)

عند تشغيل الأمر بدون خيارات، ستظهر لك لوحة تحكم تفاعلية:

```bash
rock-io run
```

#### ب. تشغيل البوتات مباشرة

لتشغيل البوتات مباشرة دون المرور بلوحة التحكم:

```bash
rock-io run --bots
# أو
rock-io run boots
```

## أوامر الأداة

| الأمر | الوصف | مثال |
| :--- | :--- | :--- |
| `rock-io setup` | تهيئة المشروع واختيار الملفات المراد تحليلها. | `rock-io setup` |
| `rock-io run` | تشغيل لوحة التحكم التفاعلية أو بدء التحليل. | `rock-io run` |
| `rock-io run boots` | تشغيل البوتات التحليلية مباشرة. | `rock-io run boots` |
| `rock-io -V` | عرض إصدار الأداة. | `rock-io -V` |
| `rock-io --help` | عرض قائمة المساعدة والأوامر. | `rock-io --help` |

## معمارية البوتات (Bots Architecture)

تعتمد الأداة على وحدة تحليل مركزية (`src/bots/analyzer.js`)، والتي يمكن توسيعها بسهولة لإضافة "بوتات" جديدة أو قواعد فحص مخصصة للغات برمجية مختلفة (مثل JavaScript، Python، HTML، إلخ).

**مبدأ عمل البوت:**
1.  يقرأ البوت محتوى الملف محليًا.
2.  يطبق مجموعة من قواعد التعبير النمطي (Regex) أو تحليل شجرة بناء الجملة (AST - مستقبلاً) للبحث عن أنماط محددة.
3.  في حال اكتشاف مشكلة، يتم تسجيلها كـ "خطأ محتمل" (Potential Bug) دون محاولة إصلاحها، تاركًا القرار للمطور.

**مثال على اكتشاف البوت:**
*   البوت يكتشف استخدام `var` في ملف JavaScript ويقترح استخدام `let` أو `const` كأفضل ممارسة.
*   البوت يكتشف كتلة `catch` فارغة في Python أو JavaScript ويحذر من تجاهل الأخطاء بصمت.
